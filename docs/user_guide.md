# Руководство редактора контента

## 1. Структура каталога `content/data`
Все сообщения бота хранятся в виде **YAML‑файлов** в каталоге `content/data/`. Каждый файл описывает один узел (node) диалогового дерева.

### Минимальный шаблон узла
```yaml
id: уникальный_идентификатор   # строка, используется в навигации
type: text | image | mixed      # тип контента
# При type = text
text: |
  Привет! Это пример текста.
# При type = image (можно указать одновременно text и image)
image: path/to/image.jpg
caption: |
  Подпись к изображению (необязательно).
# Клавиатура – массив кнопок, каждая ведёт к другому узлу
keyboard:
  - label: "Кнопка 1"
    target: id_следующего_узла
  - label: "Кнопка 2"
    target: другой_узел
# Параметры управления появлением
available_from: "2025-08-22"   # ISO‑date, если дата в прошлом – узел видим
feature_flag: "beta"           # произвольный флаг, проверяется через переменную окружения
```

## 2. Добавление нового узла
1. Создайте файл `content/data/<имя>.yaml` со структой, приведённой выше.
2. Убедитесь, что **id уникален** во всей иерархии.
3. Добавьте кнопку‑ссылку в родительский узел, указав `target` равным только что созданному `id`.
4. Коммитьте файл – при включённом `watchdog` (по умолчанию в `content_repository.py`) бот автоматически подхватит изменения без перезапуска.

## 3. Управление прогрессивным выпуском
- **available_from** – дата, после которой узел будет доступен пользователям. Если текущая дата меньше, кнопка, указывающая на этот узел, **не будет включена** в клавиатуру.
- **feature_flag** – строковое имя флага. Если переменная окружения `FEATURE_FLAGS` содержит этот флаг (список через запятую), узел считается доступным.

### Пример использования feature flag
```yaml
feature_flag: "new_year"
```
Запуск бота с переменной `FEATURE_FLAGS=new_year,experimental` сделает узел видимым.

## 4. Валидация контента
Для локальной проверки используйте скрипт:
```bash
python -m scripts.validate_content
```
Он проверит:
- корректность YAML‑синтаксиса;
- уникальность `id`;
- отсутствие «мертвых» ссылок (кнопка указывает на несуществующий `target`).

## 5. Пример полного узла (из реального бота)
```yaml
id: schedule_root
type: text
text: |
  Выберите день мероприятия:
keyboard:
  - label: "Первый день"
    target: day_1
  - label: "Второй день"
    target: day_2
available_from: "2025-08-22"
```

---
**Совет:** Если хотите быстро скопировать схожие узлы, используйте `cp` и меняйте только нужные поля. Всё остальное будет автоматически обработано системой.
