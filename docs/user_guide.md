# Руководство редактора контента

## 1. Структура каталога `content/data`
Все сообщения бота хранятся в виде **YAML‑файлов** в каталоге `content/data/`. Каждый файл описывает один узел (node) диалогового дерева.

### Новый тип узла – `menu`
`menu`‑узел объединяет **контент** (внутри `PostNode`) и **клавиатуру**. В отличие от прежних `post`‑узлов, клавиатура теперь объявляется только в `MenuNode`.

```yaml
id: main                 # уникальный id, обязательный для корневого узла
content:                  # вложенный PostNode (только текст/медиа)
  media:
    type: text
    text: Добро пожаловать в бот!
  available_from: "2025-01-01"
  available_to: "2025-12-31"

keyboard:                 # обязательное поле – массив массивов кнопок
  - - label: "События"
      target: events_root
  - - label: "Настройки"
      target: settings_root
flags:                    # опциональные флаги управления автокнопками
  - is_back: true        # добавить кнопку «Назад», если есть история (по‑умолчанию true)
  - is_main: true        # добавить кнопку «Главное меню» (по‑умолчанию true)
```

### Примечание
* **Клавиатура** теперь задаётся в `MenuNode` и состоит из массивов массивов (строк) кнопок, каждая из которых имеет поля `label`, `target`, `is_back` и `is_main` (по‑умолчанию `false`).
* При генерации меню система автоматически добавит кнопку **«Назад»**, если в `NavigationContext.history` есть предыдущий узел, и кнопку **«Главное меню»**, если `is_main` = `true`.
* Чтобы отключить автоматические кнопки, укажите в `flags` соответствующее значение `false`.
* Корневая нода обязана иметь `id: "main"`.

## 2. Добавление нового узла
1. Создайте файл `content/data/<имя>.yaml` со структой, приведённой выше.
2. Убедитесь, что **id уникален** во всей иерархии.
3. Коммитьте файл – при включённом `watchdog` (по умолчанию в `content_repository.py`) бот автоматически подхватит изменения без перезапуска.

## 3. Контент

Внутри `PostNode` (а также в поле `content` у `MenuNode`) могут использоваться следующие типы контента:

| **type** | **Описание** | **Обязательные поля** | **Опциональные поля** |
|----------|--------------|-----------------------|-----------------------|
| `text`   | Текстовое сообщение. | `text: |` (многострочный) | `available_from`, `available_to`, `flags` |
| `photo`  | Фотография с подписью. | `url: <url>` | `description`, `available_from`, `available_to`, `flags` |
| `video`  | Видео‑файл с подписью. | `url: <url>` | `description`, `available_from`, `available_to`, `flags` |

`PostNode` может содержать список медиа‑элементов в поле `media`. Каждый элемент списка – это один из вышеописанных типов (`PhotoNode`, `VideoNode` или `TextNode`). Пример структуры `PostNode` с несколькими медиа:

```yaml
id: example_post
media:
  - type: photo
    url: https://example.com/image.jpg
    description: "Красивый вид"
  - type: video
    url: https://example.com/video.mp4
    description: "Видео‑демонстрация"
  - type: text
    text: |
      Дополнительный описательный текст.
available_from: "2025-01-01"
available_to: "2025-12-31"
flags: []
```

* `available_from` и `available_to` задают временные ограничения, когда данный узел будет доступен пользователю.
* `flags` – список произвольных строк, которые могут использоваться для условного отображения узла (например, feature‑флаги). В текущей версии они оставлены пустыми, но могут расширяться в будущем.

## 4. Валидация контента
Для локальной проверки используйте скрипт:
```bash
python -m scripts.validate_content
```
Он проверит:
- корректность YAML‑синтаксиса;
- уникальность `id`;
- отсутствие «мертвых» ссылок (кнопка указывает на несуществующий `target`).

## 5. Пример полного узла (из реального бота)

Ниже показан типичный `menu`‑узел, который включает контент‑часть (`PostNode`), клавиатуру и опциональные параметры. Такой файл обычно помещается в `content/data/`.

```yaml
id: main                           # корневая нода – обязательный id

content:                           # вложенный PostNode – можно использовать любые типы медиа
  media:
    - type: photo
      url: https://example.com/welcome.jpg
      description: "Приветственное изображение"
    - type: text
      text: |
        Добро пожаловать в наш бот! Здесь вы найдете актуальную информацию о мероприятиях.
  available_from: "2025-01-01"
  available_to: "2025-12-31"
  flags: []

keyboard:                          # обязательная клавиатура (массив массивов строк)
  - - label: "События"
      target: events_root
      is_back: false
      is_main: false
  - - label: "Настройки"
      target: settings_root
      is_back: false
      is_main: false

flags:                             # управление автокнопками (по‑умолчанию true)
  - is_back: true                 # добавить кнопку «Назад», если есть история
  - is_main: true                 # добавить кнопку «Главное меню»
```

**Что важно обратить внимание:**
- Поле `content` полностью описывает пост‑контент (в примере – фото + текст). Можно использовать любой набор медиа‑элементов.
- В `keyboard` каждая кнопка имеет `label` и `target`. Поля `is_back` и `is_main` могут быть заданы явно, но обычно их не указывают – система добавит их автоматически в зависимости от `flags`.
- `flags` управляют тем, будет ли система автоматически добавлять кнопки «Назад» и «Главное меню». Если их установить в `false`, автокнопки не появятся.
- Параметры `available_from`/`available_to` позволяют ограничить время жизни узла.
- При необходимости можно добавить пользовательские `flags` (например, `beta`, `promo`).

**Совет:** При копировании узла меняйте только `id`, `target`‑ы и, при необходимости, медиа‑контент. Всё остальное (автокнопки, ограничения по дате) можно оставить без изменения.
