# Руководство редактора контента

## Структура каталога `content`
### Основные данные
Все сообщения бота хранятся в виде **YAML‑файлов** в каталоге `content/data/`.
Каждый файл описывает один узел диалогового дерева.
Отдельный узел может содержать текст, фото, видео или группу таких объектов.
Обычно узел это объект **Post** или **PostGroup**.
Группа постов помогает отображать сложные сценарии, когда возможности одного поста ограничены.
Например, в одном посте может быть медиа группа с фото и видео, а в другом текст с навигационной клавиатурой.
В **Post** может быть реализована клавиатура, в которой указывается `target` - это ссылка на другой узел, название файла.

#### Обязательные ноды
* `id: "main"` - Нода с меню, которая отображается с командной Старт
* `id: "help"` - Нода для команды Помощь
* `id: "error"` - Нода на случай ошибок в работе бота
* `id: "retry"` - Нода для ответа, если пользователь совершил неожиданный ввод


### Изображения
Изображения хранятся в папке `content/photo/`. Они могут образовывать подкатегории.
В узлах Медиа объекты должны иметь два поля `type` и `local_path`, дополнительно может быть `description`
`type` всегда всегда равен `photo`, а `local_path` - это ссылка на фото с указанием относительного пути от `.../photo/`.
Пример:
```yaml
type: photo
local_path: teachers/igor_olya.jpg
description: >
  description
```

### Видео
Располагаются в папке `content/video/` и подвергаются тем же правилам, что и изображения в отношении полей `type` и `local_path`.
`type` всегда равен `video`, а `local_path` показывает, где найти файл в папке `.../video/`.
Добавляется поле `vk_url`, которое позволяет отправлять видео в ВК без сложных авто загрузок.
```yaml
type: video
local_path: locations/sds.mp4
vk_url: https://vkvideo.ru/video-14675331_456239118
description: >
  some words
```

## Описание узла

### Post Group
Служит для объединения **Постов** в одну группу. Список постов без дополнительного функционала.
Обязательные поля:
- `id` - уникальный id
- `type` - тип объекта, всегда `post_group`
- `posts` - список из `Post` объектов
```yaml
id: main
posts:
  - one_post
  - second_post
```

### Post
Главный объект при описании узла. Объединяет медиа, клавиатуру и настройки. Поля:
- `id` - уникальный id
- `type` - всегда `post`
- `media` - медиа группа или лишь один медиа объект
- `keyboard` - клавиатура
- `flags` - дополнительные настройки

#### Media
Хранит контент для постов. Сейчас возможны три варианта контента: текст, фото, видео.
Медиа может содержать как одну единицу контента, так и **Медиа группу** - это список объектов.
**Медиа группа** может содержать только фото и видео объекты.
А также в посте не может быть одновременно **Медиа Группы** и **Клавиатуры**.

Структура **Фото** и **Видео** подробно описана выше. **Текст** имеет лишь два поля: `type`, `text`.

```yaml
media:
  type: text
  text: Hello World
```
```yaml
media:
  - type: photo
    local_path: location/party.jpg
    description: Let's have a party!
  - type: video
    local_path: location/party.mp4
    description: Let's go!
```

#### Keyboard
Помогает организовывать навигацию между разными узлами. Пишется в виде списка списков: Клавиатура → Строка → Кнопка.
Возможные поля **Кнопки**:
- `text` - Отображаемое название кнопки
- `target` - Ссылка на другой узел. Может быть как полным названием другого файла, так и без расширения
- `type` - Тип кнопки, по умолчанию "default". В данной версии используется только для служебных кнопок. Возможны значения "main_menu" или "back"

##### Добавляемые кнопки
Бот в ходе работы добавляет кнопки **Главное меню** и **Назад** на основании активности пользователя.
Кнопка **Главное меню** добавляется если пользователь имеет историю длиной более 2-х узлов, а **Назад** добавляется при
длине больше одного узла. Это поведение можно настроить в `flags`.

```yaml
keyboard:
  - - text: "Цены"
      target: prices
    - text: "Музыкальные группы"
      target: music_groups
  - - text: "Локации"
      target: locations
```

#### Flags
Настройки поста.
- `is_back` - добавить кнопку "Назад", по умолчанию true
- `is_main` - добавить кнопку "Главное меню", по умолчанию true
- `build` - добавлять кнопки, по умолчанию true

```yaml
flags:
  is_back: true
  is_main: true
  build: true
```

#### Полный пример Медиа узла
```yaml
id: error
type: post
media:
  type: text
  text: |
    😔 Произошла техническая ошибка
keyboard:
  - - text: "Связь с организаторами"
      target: contacts
```
```yaml
id: media_group
type: post
media:
  - type: video
    local_path: locations/open.mp4
    vk_url: https://vk.com/clip-213821730_456239145
  - type: photo
    local_path: locations/open.JPG
  - type: photo
    local_path: locations/open_map.jpg
flags:
  build: false
```

## Валидация контента
Для локальной проверки используйте скрипт:
```bash
make check-content
```
Он проверит:
- корректность YAML‑синтаксиса;
- уникальность `id`;
- отсутствие «мертвых» ссылок (кнопка указывает на несуществующий `target`).
